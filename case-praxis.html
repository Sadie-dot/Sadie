Study: Guiding client to a simpler design that fit their needs by asking about goals

User flow inefficiencies were limiting business growth.
How do you redesign when clients can't articulate what they want?

  • The existing user flows had friction that prevented caregivers from completing critical tasks efficiently.
  • Client stakeholders described symptoms — not root causes — making it hard to determine design direction.
  • A narrow product pipeline meant any redesign had to be precise; there was no room for guesswork.

My role

UX Researcher & Designer

Project

Web app to track health metrics of children

Client

PRAXIS — New York-based start-up
Narrow product pipeline

PRAXIS

Step 1 | Problem

Understanding the real objectives behind the user flow complaints.

Rather than jumping to wireframes, I started by facilitating goal-setting sessions with the client team to separate business outcomes from interface preferences.

  • Mapped existing user flows to surface where drop-offs and errors occurred most frequently.
  • Conducted stakeholder interviews to distinguish what clients wanted from what users needed.
  • Identified that the core issue was task sequencing, not visual design.
User flow mapping — existing state

Existing flow diagram revealing friction points in the caregiver task sequence

Step 2 | Definition

Asking about goals, not features, unlocked the design direction.

By reframing client feedback as goal statements, I was able to translate vague requests into testable design criteria.

  • Ran a structured workshop where stakeholders ranked outcomes by importance rather than listing features.
  • Produced a simplified information architecture that reduced cognitive load for caregivers.
  • Validated direction with rapid prototype walkthroughs before committing to full designs.
Goal-mapping workshop output

Prioritised user goals mapped to redesigned navigation structure

Step 3 | Deliver

A simpler flow that matched what users actually needed to accomplish.

The redesigned flows reduced task completion steps and aligned the product roadmap with verified user needs.

  • Delivered annotated wireframes and a revised user journey aligned to business objectives.
  • Created a design rationale document for the client to use with their engineering team.
  • Recommended a phased delivery plan that fit within the start-up's narrow product pipeline.

A goal-first process delivered a cleaner product aligned to both user needs and client business objectives.

  • Client was able to communicate design intent clearly to engineers using the rationale documentation.
  • Simpler task flows reduced projected support burden for the start-up's lean team.
  • The goal-mapping approach became a repeatable framework for subsequent product decisions.
